Services

Angular services are substitutable objects that are wired together using dependency injection (DI). You can use services to organize and share code across your app.

Angular services are:

  • Lazily instantiated – Angular only instantiates a service when an application component depends on it.
  • Singletons – Each component dependent on a service gets a reference to the single instance generated by the service factory.

Angular offers several useful services (like ${symbol_dollar}http), but for most applications you&${symbol_pound}39;ll also want to create your own.

Note: Like other core Angular identifiers, built-in services always start with ${symbol_dollar} (e.g. ${symbol_dollar}http).

Creating Services

Application developers are free to define their own services by registering the service&${symbol_pound}39;s name and service factory function, with an Angular module.

The service factory function generates the single object or function that represents the service to the rest of the application. The object or function returned by the service is injected into any component (controller, service, filter or directive) that specifies a dependency on the service.

Registering Services

Services are registered to modules via the Module API. Typically you use the Module${symbol_pound}factory API to register a service:

myModule.factory(&${symbol_pound}39;serviceId&${symbol_pound}39;, function() {
  var shinyNewServiceInstance;
  // factory function body that constructs shinyNewServiceInstance
  return shinyNewServiceInstance;

Note that you are not registering a service instance, but rather a factory function that will create this instance when called.

orderBy

  1. - filter in module ng

Orders a specified array by the expression predicate. It is ordered alphabetically for strings and numerically for numbers. Note: if you notice numbers are not being sorted correctly, make sure they are actually being saved as numbers and not strings.

Usage

In HTML Template Binding

{{ orderBy_expression | orderBy : expression : reverse}}

In JavaScript

${symbol_dollar}filter(&${symbol_pound}39;orderBy&${symbol_pound}39;)(array, expression, reverse)

Arguments

Param Type Details
array Array

The array to sort.
expression function(*)stringArray.<(function(*)|string)>=

A predicate to be used by the comparator to determine the order of elements.

Can be one of:

  • function: Getter function. The result of this function will be sorted using the <, =, > operator.
  • string: An Angular expression. The result of this expression is used to compare elements (for example name to sort by a property called name or name.substr(0, 3) to sort by 3 first characters of a property called name). The result of a constant expression is interpreted as a property name to be used in comparisons (for example "special name" to sort object by the value of their special name property). An expression can be optionally prefixed with + or - to control ascending or descending sort order (for example, +name or -name). If no property is provided, (e.g. &${symbol_pound}39;+&${symbol_pound}39;) then the array element itself is used to compare where sorting.

  • Array: An array of function or string predicates. The first predicate in the array is used for sorting, but when two items are equivalent, the next predicate is used.

    If the predicate is missing or empty then it defaults to &${symbol_pound}39;+&${symbol_pound}39;.
reverse
(optional)
 boolean

Reverse the order of the array.

Returns

Array

Sorted copy of the source array.

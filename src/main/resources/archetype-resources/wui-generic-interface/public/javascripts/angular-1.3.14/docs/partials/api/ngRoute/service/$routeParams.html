${symbol_dollar}routeParams

  1. - service in module ngRoute

The ${symbol_dollar}routeParams service allows you to retrieve the current set of route parameters.

Requires the ngRoute module to be installed.

The route parameters are a combination of ${symbol_dollar}location&${symbol_pound}39;s search() and path(). The path parameters are extracted when the ${symbol_dollar}route path is matched.

In case of parameter name collision, path params take precedence over search params.

The service guarantees that the identity of the ${symbol_dollar}routeParams object will remain unchanged (but its properties will likely change) even when a route change occurs.

Note that the ${symbol_dollar}routeParams are only updated after a route change completes successfully. This means that you cannot rely on ${symbol_dollar}routeParams being correct in route resolve functions. Instead you can use ${symbol_dollar}route.current.params to access the new route&${symbol_pound}39;s parameters.

Dependencies

// Given:
// URL: http://server.com/index.html${symbol_pound}/Chapter/1/Section/2?search=moby
// Route: /Chapter/:chapterId/Section/:sectionId
//
// Then
${symbol_dollar}routeParams ==> {chapterId:&${symbol_pound}39;1&${symbol_pound}39;, sectionId:&${symbol_pound}39;2&${symbol_pound}39;, search:&${symbol_pound}39;moby&${symbol_pound}39;}

form.FormController

  1. - type in module ng

FormController keeps track of all its controls and nested forms as well as the state of them, such as being valid/invalid or dirty/pristine.

Each form directive creates an instance of FormController.

Methods

  • ${symbol_dollar}rollbackViewValue();

    Rollback all form controls pending updates to the ${symbol_dollar}modelValue.

    Updates may be pending by a debounced event or because the input is waiting for a some future event defined in ng-model-options. This method is typically needed by the reset button of a form that uses ng-model-options to pend updates.

  • ${symbol_dollar}commitViewValue();

    Commit all form controls pending updates to the ${symbol_dollar}modelValue.

    Updates may be pending by a debounced event or because the input is waiting for a some future event defined in ng-model-options. This method is rarely needed as NgModelController usually handles calling this in response to input events.

  • ${symbol_dollar}addControl();

    Register a control with the form.

    Input elements using ngModelController do this automatically when they are linked.

  • ${symbol_dollar}removeControl();

    Deregister a control from the form.

    Input elements using ngModelController do this automatically when they are destroyed.

  • ${symbol_dollar}setValidity();

    Sets the validity of a form control.

    This method will also propagate to parent forms.

  • ${symbol_dollar}setDirty();

    Sets the form to a dirty state.

    This method can be called to add the &${symbol_pound}39;ng-dirty&${symbol_pound}39; class and set the form to a dirty state (ng-dirty class). This method will also propagate to parent forms.

  • ${symbol_dollar}setPristine();

    Sets the form to its pristine state.

    This method can be called to remove the &${symbol_pound}39;ng-dirty&${symbol_pound}39; class and set the form to its pristine state (ng-pristine class). This method will also propagate to all the controls contained in this form.

    Setting a form back to a pristine state is often useful when we want to &${symbol_pound}39;reuse&${symbol_pound}39; a form after saving or resetting it.

  • ${symbol_dollar}setUntouched();

    Sets the form to its untouched state.

    This method can be called to remove the &${symbol_pound}39;ng-touched&${symbol_pound}39; class and set the form controls to their untouched state (ng-untouched class).

    Setting a form controls back to their untouched state is often useful when setting the form back to its pristine state.

  • ${symbol_dollar}setSubmitted();

    Sets the form to its submitted state.

Properties
